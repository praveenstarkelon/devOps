  |
  +--> [ name = "my-cluster" ]
  |
  +--> [ role_arn = "EKS Cluster IAM Role" ]
    |    +--> Trust: eks.amazonaws.com
    |    +--> Policy: AmazonEKSClusterPolicy
  |
  +--> [ version = "1.30" ]
  +--> [ vpc_config ]
        +--> subnet_ids = ["subnet-123", ...]
        +--> security_group_ids = [...]

1.3 Best Practices

  • Cluster role से सिर्फ cluster manage करें, वहां production workloads ना रखें
  • VPC/subnets networking हमेशा private+public दोनों तरह की बनाएं
  • Policies की least privilege approach follow करें
